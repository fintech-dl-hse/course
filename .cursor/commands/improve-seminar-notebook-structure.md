# Улучшение структуры ноутбука семинара

Примени эту команду к открытому ноутбуку семинара (`seminars/**/*.ipynb`). Приведи уровни заголовков и форматирование в соответствие с иерархией и правилами из `.cursor/rules/seminars-overview.mdc`.

## Иерархия заголовков

1. **Один H1** в начале — название семинара, например: `# Семинар N: Название темы`
   - Если первый заголовок — «Структура» или план, оформи как подзаголовок: `### Структура` под общим H1.

2. **## (H2)** — основные блоки семинара:
   - Recap (если есть): `## Recap: регуляризация` (или соответствующая тема)
   - Крупные темы: «Работа с изображениями», «Свертки», «Pooling и рецептивное поле», «TorchVision Models», «Блиц», «Домашки»

3. **### (H3)** — подразделы внутри блока:
   - Примеры: Dropout, Optimizers, BatchNorm2d, «Попробуем применить свертку», «Свертки в PyTorch», CNNModel, Pooling, Global Pooling, Рецептивное поле, Residual Block, «Как применить модель», «Что дальше»

4. **#### (H4)** — только для вопросов в формате из правил (см. ниже).

## Вопросы (blitz и по ходу семинара)

- Каждый вопрос оформи как **отдельная ячейка** с заголовком уровня 4.
- Формат строго: `#### ❓ **Вопрос**: текст вопроса?`
- Замени варианты вроде `**Вопрос:**`, `**Вопрос!**`, `### **Вопрос!**` на этот формат.
- В секции «Блиц» все пункты, которые являются вопросами, переведи в `#### ❓ **Вопрос**: ...`

## Дополнительные правки форматирования

- **Списки в markdown:** используй `-` для маркированных списков; для перечислений с подчёркиванием терминов — `**SGD**`, `**Adam**` и т.д.
- **Стрелки:** предпочтительно `→` вместо `->` в тексте.
- **Опечатки:** исправь типичные («лекци» → «лекции», «тк» → «т.к.», «разные»/«разнные» по смыслу, «доп место» → «доп. место», «проистранственную» → «пространственную», «испорититься» → «испортиться»).
- **Псевдо-заголовки в коде:** если в ячейке с кодом первая строка выглядит как markdown-заголовок (например `## копипаста`), удали её из кода и при необходимости добавь перед ячейкой отдельную markdown-ячейку с заголовком `### ...`.
- **Домашки:** заголовок оформи как `## Домашки`; список — через `-`, не `*`.
- **Без trailing spaces** во всех изменённых ячейках.

## Порядок действий

1. Прочитай `.cursor/rules/seminars-overview.mdc` и текущий ноутбук.
2. Составь кратко текущую карту заголовков (какие #, ##, ### есть и где).
3. Приведи уровни к иерархии выше: один H1, блоки ##, подразделы ###, вопросы ####.
4. Добавь недостающие подзаголовки (например, «Что происходит под капотом?» → `### Что происходит под капотом?`, блок про изображения → `### RGB-каналы` и т.д.), если по смыслу они нужны.
5. Все вопросы переведи в формат `#### ❓ **Вопрос**: ...`.
6. Примени перечисленные правки форматирования и проверь отсутствие trailing spaces.

Редактируй ячейки через замены в JSON ноутбука (StrReplace) или через EditNotebook — как удобнее для однозначных совпадений. Не меняй код в ячейках, только markdown и структуру.
